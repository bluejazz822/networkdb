Network CMDB API Documentation

Comprehensive RESTful API for managing network infrastructure resources

Quick Start

This API provides complete CRUD operations for network resources including VPCs, Transit Gateways, Customer Gateways, and VPC Endpoints.

VPCs

Manage Virtual Private Clouds with full lifecycle operations

Transit Gateways

Control Transit Gateway resources and routing configurations

Customer Gateways

Manage Customer Gateway connections for VPN setups

VPC Endpoints

Configure private connections to AWS services

Features

  • Full CRUD Operations: Create, read, update, and delete all resource types
  • Advanced Filtering: Filter by region, state, environment, owner, and more
  • Pagination & Sorting: Efficient data retrieval with customizable pagination
  • Bulk Operations: Create or delete multiple resources in a single request
  • Validation: Comprehensive input validation with detailed error messages
  • Audit Logging: All operations are logged with user context
  • AWS Integration: Native support for AWS resource IDs and patterns

Base URLs

  • Development: http://localhost:3001/api
  • Production: https://api.network-cmdb.example.com/api

Authentication

Authentication is required for all endpoints. Include your API token in the request headers:

Authorization: Bearer <your-api-token>

Or use API Key authentication:

X-API-Key: <your-api-key>
